## v1.14.4 — Discovery + Identità visiva

### Punto di partenza

L'utente ha aperto la sessione con una segnalazione: *"non vedo più la wiki nel menu laterale"*. Diagnosi: dalla **v1.12.0** (riorganizzazione sidebar) la configurazione delle sorgenti era finita **dentro l'expander `⚙️ Configurazione`**, chiuso di default. La feature esisteva, ma era invisibile a colpo d'occhio.

### Decisione UX

Considerate due opzioni:
- **A.** Expander dedicato per la KB, aperto di default, toggle "Usa KB" mantenuto
- **B.** Eliminare il toggle e rendere la KB sempre attiva

Scelta **A** perché il toggle è anche gate del warning privacy su Cloud provider; toglierlo per "guadagnare un click" non vale il rischio. Aggiunta anche una **caption con elenco delle sorgenti disponibili** quando il toggle è off, per amplificare la discoverability.

### Identità visiva nei launcher (parte 2 di v1.14.4)

L'utente ha aggiunto `deepaiug-logo.png` e `deepaiug-logo.ico` in root. Sostituite le icone di default OS nei collegamenti creati dagli installer:

| OS | Implementazione |
|---|---|
| 🐧 Linux | `Icon=` nel `.desktop` (PNG nativo) |
| 🪟 Windows | `oLink.IconLocation` nel VBScript (`.ico`, con check di esistenza file) |
| 🍎 macOS | Nuovo wrapper `DeepAiUG.app` con `.icns` generato al volo da `sips`+`iconutil`, `.command` esistente come fallback CLI |

### Release v1.14.4

Promossa come **Latest** sostituendo v1.14.3, con descrizione narrativa pubblicata via web UI GitHub (no `gh` CLI installato).

## v1.15.0 — Embedding multilingua e qualità retrieval

### Trigger della discussione

Domanda dell'utente: *"a volte i documenti non sembrano rispondere bene. Se usassimo un modello LLM di Ollama sarebbe meglio?"*

Risposta articolata: il sistema usa **`all-MiniLM-L6-v2`** silenziosamente come default ChromaDB — modello principalmente inglese, su corpus italiano produce retrieval mediocre. Ollama non è la prima soluzione (richiede cambi grossi, e su CPU mobile non è veloce); soluzione più efficace e meno invasiva è **cambiare modello sentence-transformers a uno multilingua**.

### Modello scelto

**`intfloat/multilingual-e5-small`** — 117M params, 384-dim, multilingua nativo, top-3 MTEB italiano, ~118 MB download. Drop-in dimensionalmente con MiniLM-L6, override possibile via env var `DEEPAIUG_EMBEDDING_MODEL`.

### Architettura della modifica

- Nuovo modulo `rag/embeddings.py` — fonte di verità per embedding
- Strategia prefix corretta: `passage:` per indicizzazione, `query:` per ricerca
- Migrazione automatica delle collection legacy via metadata `embedding_model`
- Fallback graceful se sentence-transformers non disponibile

### Test in produzione e iterazioni

La sessione si è espansa per coprire i bug emersi al primo test reale.

#### Iterazione 1 — Indicizzazione "rapidissima"

L'utente segnala che l'indicizzazione finisce in pochi secondi. Diagnosi: **session_state stale di Streamlit**. Il vecchio `kb_manager` v1.14.4 era cached e usava il default MiniLM. Riavvio completo di Streamlit ha risolto.

#### Iterazione 2 — Migration check incompleto

Verifica CLI mostra `embedding_model: <NON IMPOSTATO>` su collection esistente. **Bug nel migration check**: la condizione `stored_tag and stored_tag != active_tag` lasciava passare collection senza tag (legacy). Fix: `stored_tag != active_tag` — anche `None` scatena il reset.

Aggiunto **indicatore UI del modello attivo** nella sezione Statistiche KB della sidebar — debug visivo immediato.

#### Iterazione 3 — Banner stima tempo

L'utente nota che mancava il warning "operazione lunga" che ricordava dalle versioni precedenti. Era in `_show_load_warning` (caricamento conversazioni con KB), ma non nel `_sync_source` (indicizzazione manuale). Aggiunto banner in `_sync_source` che copre tutti i 6 entry point (vault, cartella locale, MediaWiki, DokuWiki, custom + YAML).

#### Iterazione 4 — Calibrazione sui tempi reali

Indicizzazione vera del vault Obsidian dell'utente: **966 file → 18 minuti** su i7-8565U mobile. Stima precedente diceva ~6 min (sottostima 2.8×). Ricalibrata tabella `_TIME_PER_FILE_SECONDS`:

| Modello | s/file (cal. v1.15.0) |
|---|---|
| `intfloat/multilingual-e5-small` | **1.10** *(misurato)* |
| `intfloat/multilingual-e5-base` | 3.00 *(scalato)* |
| `intfloat/multilingual-e5-large` | 5.50 *(scalato)* |
| `paraphrase-multilingual-MiniLM-L12-v2` | 0.80 |
| `chromadb_default` (MiniLM-L6 ONNX) | 0.10 |

#### Iterazione 5 — Allineamento dei due banner

`_show_load_warning` in `conversations.py` usava ancora un fattore hardcoded `0.4 s/file` (residuo v1.13.5). Allineato a `get_seconds_per_file()` centralizzato → cambiare modello aggiorna entrambi i banner automaticamente.

#### Iterazione 6 — Bugfix Remote host

L'utente prova a caricare una chat archiviata con KB associata, in modalità Remote host. Il sistema blocca con messaggio *"Non può essere caricata con un provider cloud per motivi di privacy"*. **Bug**: la variabile `is_cloud` era definita come `!= "Local (Ollama)"` → catturava sia Cloud sia Remote.

Fix architettonico: `is_cloud = == "Cloud provider"` (definizione **per affermazione del non-trusted**, non per negazione del trusted).

## Pattern e principi emersi

### 1. Discoverability ≠ click count
Spostare un blocco UI da expander chiuso a aperto + caption con elenco opzioni risolve un problema di "non lo vedo" senza toccare la logica di gating.

### 2. Una modifica RAG, due collection
DeepAiUG ha due collection ChromaDB: `wiki_knowledge_base` e `deepaiug_chat_kb`. Centralizzare la logica embedding in un nuovo modulo (`rag/embeddings.py`) ha pagato — due call site, una sola fonte di verità.

### 3. Migration conservativo + diagnostica visibile
Auto-migrazione solo per collection con tag mismatch è risultato troppo permissivo (collection legacy senza tag passavano). Fix: trattare anche `None` come "tag diverso". Diagnostica UI (indicatore embedding model in sidebar) rende il bug rilevabile.

### 4. Stime oneste, calibrate sul reale
Stima a tavolino vs misurazione reale: la differenza era 3×. Aggiornare le tabelle quando si hanno dati veri vince sempre. **Stesso fattore in tutti i punti UI** (DRY): un solo helper `get_seconds_per_file()` consumato da banner di indicizzazione e banner di caricamento conversazione.

### 5. Privacy: definire per affermazione del non-fidato
`is_cloud = != Local` cattura erroneamente i provider trusted come Remote host. `is_cloud = == Cloud` è esplicito e si scala a nuovi provider trusted senza modifiche.

### 6. Helper + EmbeddingFunction separati
ChromaDB chiama `embedding_function` solo per le query. Per i documenti pre-computiamo separatamente con prefix `passage:`. Pattern pulito che rispetta il contratto ChromaDB e usa il prefix corretto in ogni caso.

### 7. Fallback graceful
Se `sentence-transformers` non disponibile: warning a console + fallback a default ChromaDB. Sistema funzionante con qualità ridotta, niente crash. Importante per `git clone` senza `pip install` e per sistemi air-gapped.

### 8. Maintenance mode ≠ "niente patch"
v1.14.3 era stata dichiarata "ultimo rilascio del ciclo v1.x". v1.14.4 e v1.15.0 hanno mostrato che maintenance mode non significa congelamento totale — correzioni di discoverability, identità visiva e qualità retrieval sono atterrate senza rompere il principio "no nuove feature, solo correzioni e affinamenti".

### 9. Riavviare Streamlit completamente dopo modifiche significative
`session_state` non viene resettato dai `Rerun`. Per modifiche a oggetti cached (come `kb_manager`), serve **Ctrl+C + relaunch** del processo. Importante per evitare bug "fantasma" dovuti a oggetti vecchi nel session.

## Lezioni operative finali

1. **Riavviare Streamlit completamente** dopo modifiche al codice di moduli importati da `session_state` (es. `kb_manager`).

2. **Verificare il modello attivo via CLI** dopo aggiornamenti embedding:
   ```bash
   ./venv/bin/python3 -c "
   import chromadb
   client = chromadb.PersistentClient(path='knowledge_base/vectorstore')
   coll = client.get_collection('wiki_knowledge_base')
   print('embedding_model:', (coll.metadata or {}).get('embedding_model'))
   print('chunks:', coll.count())
   "
   ```

3. **Calibrare sui dati reali**: una misurazione vera batte una stima a tavolino. Aggiornare le tabelle quando arrivano nuovi punti dati.

4. **Documentare i bugfix architetturali post-tag**: il valore della doc è cumulativo, il tag fissa uno snapshot ma il SPEC su `main` può crescere oltre lo snapshot.

5. **Test in produzione su provider diverso da quello di sviluppo**: il bug Remote host è emerso solo provando a caricare una chat archiviata in modalità Remote — flow non testato durante lo sviluppo solo Local.
